# Revision Control

Revision Control is a local study-strategy web app. It helps a student identify weak topics, generate adaptive revision sprints, run panic-mode triage, track confidence vs actual performance, and turn pasted materials into study targets.

## Project layout

- `app.py`
  Starts the local server.
- `chief_of_staff/`
  Backend code for the study engine, storage, and HTTP server.
- `static/`
  Frontend UI, styles, and browser-side interactions.
- `tests/`
  Unit tests for the study strategy service.
- `data/`
  Local runtime data and local-only config files. This folder is gitignored except for `.gitkeep`.

## Local setup

1. Clone the repository.
2. Go into the project folder.
3. Start the app:

```bash
python3 app.py
```

4. Open:

```text
http://127.0.0.1:8000
```

The app starts with a seeded `kennethlemuel` account. Study data is scoped per user in `data/users/<user-id>/study_state.json` so another registered account does not see Kenneth's materials or plans.

## Optional email reset setup

Password reset links are generated by the app. To send them through email, set these environment variables before starting the server:

```powershell
$env:STUDY_BUDDY_SMTP_HOST="smtp.gmail.com"
$env:STUDY_BUDDY_SMTP_PORT="587"
$env:STUDY_BUDDY_SMTP_USER="your-email@gmail.com"
$env:STUDY_BUDDY_SMTP_PASSWORD="your-app-password"
$env:STUDY_BUDDY_SMTP_FROM="your-email@gmail.com"
python app.py
```

For Gmail, use a Google app password instead of your normal account password. If these variables are not set, the app still creates a reset link locally and reports that email delivery is not configured.

## Optional local LLM setup

The current app works without an LLM. If you later want to connect a local model, keep that configuration in a local file such as:

`data/local_llm_config.json`

This file is gitignored and stays on your machine.

## Optional Spotify setup

Cram Mode can connect to Spotify for browser playback. Put Spotify credentials in a local ignored file:

`data/spotify_config.json`

Use this shape:

```json
{
  "client_id": "your-client-id",
  "client_secret": "your-client-secret",
  "redirect_uri": "http://127.0.0.1:8000/spotify/callback",
  "scopes": "streaming user-read-email user-read-private user-read-playback-state user-modify-playback-state user-read-currently-playing playlist-read-private playlist-read-collaborative"
}
```

Add the same redirect URI in the Spotify Developer Dashboard. After a user connects, the app stores Spotify tokens in `data/spotify_tokens.json`, which is also ignored, so they do not need to reconnect after restarting the app.
